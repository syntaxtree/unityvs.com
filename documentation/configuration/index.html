Configuration

Choose a target, such as standalone, to allow UnityVS to make Visual Studio and Unity communicate


Warning, targeting the Web player prevents socket connections, even in the Editor. This prevent us to communicate with Visual Studio to send console messages or open files.

Import the UnityVS package

You can import it like a standard asset package. It may take a restart of Unity for it to realize there's a new package. You can also import it directly when creating a new project. Alternatively, you can also click on the UnityVS package from the UnityVS folder of your start menu to import it in your project. Until Unity supports global packages, you'll have to do that in every project where you want to use UnityVS.

You need to make sure you import the package which maps to the Visual Studio version you are using. UnityVS 2010 for Visual Studio 2010, and UnityVS 2012 for Visual Studio 2012.

Open UnityVS from Unity

Alternatively, make sure the UnityVS solution files are generated, and open the solution file whose name starts with UnityVS.

Paste your license

The first time UnityVS is used, it will ask for a license. Simply paste the license you received after you purchase UnityVS.
