Troubleshooting

This page lists common support questions along with answers. You can also have a look at the list of known issues, or visit or bug tracker.

UnityVS solution won't load in Visual Studio

If Visual Studio complains that the generated csproj, unityproj and booproj are not supported, it's usually because the license is not properly installed. Use the license manager to make sure your UnityVS license is properly registred.

On Windows 8, opening the UnityVS solution in Visual Studio results in an error window asking to download the Unity target framework

UnityVS requires that you install the .net framework 3.5 on Windows 8.

Clicking on console entries in Unity won't navigate to the precise line in Visual Studio

That's a known issue, the reason is that Unity only provides us with the file name of the console entry. There's a couple of workarounds described in the known issue page.
